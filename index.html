Тема: Git

Що таке системи контролю версій та Git

Системи контролю версій (VCS) — це програмні системи, які допомагають відслідковувати зміни в коді, що дозволяє програмістам працювати разом над проектами, зберігати історію змін та відновлювати попередні версії коду. Git — одна з найбільш популярних систем контролю версій, розроблена Лінусом Торвальдсом. Git відрізняється швидкістю, надійністю та розподіленою архітектурою, що дозволяє працювати над проектами без постійного підключення до центрального сервера.

Основні можливості та переваги використання Git

  • Розподілена архітектура: Кожен користувач має свою повну копію репозиторія.
  • Швидкість роботи: Git забезпечує швидкий доступ до історії версій та гілок.
  • Гнучкість у роботі з гілками: Легке створення, злиття та видалення гілок.
  • Підтримка нелінійної історії: Можливість працювати з кількома паралельними гілками.
  • Висока надійність: Записи в репозиторії є незмінними, завдяки механізму хешування.

10 базових команд Git

  • git init - Ініціалізує новий Git-репозиторій.
  • git clone [url] - Клонує репозиторій з віддаленого сервера.
  • git add [file] - Додає файл до індексу для коміту.
  • git commit -m "[message]" - Зберігає зміни з коментарем.
  • git status - Показує статус змін у репозиторії.
  • git pull - Завантажує останні зміни з віддаленого репозиторія та зливає їх з локальною версією.
  • git push - Відправляє локальні зміни до віддаленого репозиторія.
  • git branch - Переглядає, створює або видаляє гілки.
  • git merge [branch] - Зливає вказану гілку з поточною.
  • git log - Показує історію комітів.
